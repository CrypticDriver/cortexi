# CortexI PRD — v0.2 规划

**状态：** v0.1.0 已开源发布，但**从未在真实 Mac 上验证过客户端**，属于半成品。本 PRD 目的是把它从"施工图纸"拉回"能用的产品"。

**写作日期：** 2026-07-01
**作者：** 狗蛋（诚实版，无夸大）

---

## 1. 问题与定位

**问题：** 大哥（AWS TAM，客户 Hungry/京东/美团）经常开会，需要：
- 会议记录不出本机（NDA 场景，客户数据敏感）
- 会后自动出结构化纪要，省手动整理时间
- 开会中能随时问"刚才决定了什么"

**定位：** 个人副脑工具，v1 只服务大哥一人，不是要做成对外产品。

**非目标（v1 明确不做）：**
- 多用户/多租户
- 移动端
- 与日历/IM 自动集成（先手动开会）
- 说话人识别超过"我/对方"两方（多人会议先不管）

| 模块 | 状态 | 证据 |

## 3. v0.2 目标（把"未知"变成"已知"）

唯一目标：**在大哥的真实 Mac 上，完整走一遍"开会→录音→转写→问答→总结"，产出可复现的证据（截图/录屏/真实纪要文本）。**

### 3.1 验证清单（Definition of Done）
- [ ] `install.sh` 在真机跑通，无手动兜底
- [ ] BlackHole 聚合设备建立成功，录到系统声音+麦克风
- [ ] 至少一次真实会议（或模拟对话）录音 → 转写文字可读，中文准确率主观可接受
- [ ] 说话人标签（我/对方）基本正确
- [ ] 会后总结包含真实决定/待办，无编造内容
- [ ] `build-app.sh` 打包成功，`.app` 双击可用，菜单栏图标能显示（或明确记录"图标不稳定，用网页界面"为已知限制而非bug）
- [ ] 全流程截图或录屏存档

### 3.2 已知需要顺手修的债
- [ ] CC 机器公网 IP：确认要不要正式关闭（若关闭，改用 SSM port-forward 或私网路径，避免走公网 ALB 之外还留后门）
- [ ] Session 生命周期：加自动清理（比如 30 天无访问自动删）或至少加手动清理命令
- [ ] 清掉 EC2 上堆积的 `app.py.bak-*` 和测试 session 垃圾
- [ ] 补最基本的单测（至少 server 的 session 生命周期 + auth 逻辑）
- [ ] README/NOW.md 措辞降级：删除"全链路E2E验证通过"这类夸大表述，改成实际验证范围

## 4. 路线图

- **v0.1.0**（已发布，需降级预期）— 后端可信，客户端未知
- **v0.2**（本PRD目标）— 客户端真机验证 + 已知债务清理，敢说"能用"

- **v1.0** — 大哥日常真实依赖，无需人工兜底
- **CortexII** — 加智能眼镜感知层（另立项，v1.0 稳定后再谈）

## 5. 风险与决策点（需要大哥拍板）

1. **CC机器公网IP**：关不关？关闭需要重新设计Mac↔远程的通道（改SSM tunnel或VPN），会有一次性改造成本。
2. **谁来做真机验证**：狗蛋这边是ARM Linux，无法运行macOS专属组件。需要大哥本人在Mac上跑，或者狗蛋远程操作大哥的Mac（如有远程桌面/SSH access）。
3. **v0.2 验证不通过怎么办**：如果whisper转写质量或rumps稳定性太差，是否降级方案（比如放弃原生菜单栏App，纯网页界面；或放弃本地whisper，改远程转写但需重新评估NDA约束）。
